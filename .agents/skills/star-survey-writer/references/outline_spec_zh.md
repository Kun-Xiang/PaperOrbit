# 大纲规格 — 先学结构，再配得上它（中文版）

思想承 SurveyForge 的大纲启发式与 AutoSurvey 的并行大纲合并（已授权，概念层）：综述的
骨架应当来自领域已经如何组织自己，并对照池的实际覆盖校验。与英文版 `outline_spec.md`
冲突时以英文版为准。

## Part A：导出候选结构

从广度表中被扫描的**综述**（强制行保证存在）：读各自的摘要——入池者再读其笔记——提取
它使用的组织轴：编年、方法分类学、问题设定、流水线阶段、评测范式。写 2–3 个候选大纲，
各标注其轴与所师法的综述。没有先行综述的领域把这一点写明（这本身是值得 intro 一句话的
发现），改从 `broad_scan.md` 的细分方向结构导出候选。

## Part B：合并与改编

把候选合并为一个大纲：取最贴合池内实际分布的轴，折入其他候选有而它缺的节，并针对最新
先行综述之后的变化做改编（前沿行正为此存在）。标准家具默认保留，除非读者画像另说：
introduction（范围 + why now + 先行综述缺什么）、background/preliminaries（仅新人读者）、
沿所选轴的主体节、open challenges/future directions、conclusion。

每个主体节带：一句话范围、**论文指派**（将取材的池内成员与只点名的扫描行）、目标篇幅。

## Part C：覆盖检查（过门之前）

- 池内每篇归属 ≥1 节；有未归属论文说明大纲或池有错——现在解决。
- 每个主体节 ≥2 篇池内论文支撑；不足的要么合并、要么显式标为薄覆盖缺口并列出能补它
  的论文（路由：经 `$star-refs-reviewer` 扩池）。
- 轴覆盖 `broad_scan.md` 计数过的每个细分方向，或在 introduction 的范围契约里把排除
  写成明确的取舍。

门上呈现大纲 + 覆盖表；用户随意重排、合并、改名。写 `outline.md`，带每节 `status:`
映射（`pending` / `in_progress` / `done`）——起草的续做起点。
