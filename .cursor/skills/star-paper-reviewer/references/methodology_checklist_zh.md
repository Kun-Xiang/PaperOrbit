# 方法论检查单 — 审稿人真实会提的质疑（中文版）

检查表框架承 Nyhan 的 peer-review checklist manifesto（经 lcrawfurd 的评审 skills，已
授权），为实证 ML 重写。每项都走一遍；只报告触发的、或"通过但值得一说"的。每条发现引
草稿位置**加**其背后的证据（或其缺失）。与英文版 `methodology_checklist.md` 冲突时以
英文版为准。

1. **Seeds 与方差。** 需要方差处只给单值；"improves by 0.3" 不给 seed 数；给均值不给
   sd。证据核对：`results.md` 各行实际记录了什么。single-seed 的 MVP 可以——*前提是
   句子明说*。
2. **对齐的比较。** Baseline 与方法在 backbone、训练预算、数据或调参投入上不一致；
   放出的 checkpoint 对上调过参的方法。经典拒稿理由。
3. **泄漏迹象。** 验证集既做选择又做汇报；对测试集调参；预训练与评测数据重叠；草稿
   从不解释的"好得过分"的跳变。核对 `EXPT_ANALYSIS` 报告已标记过什么。
4. **显著性措辞。** 无检验的 "significant(ly)"；噪声之内的优越性声明；不同 split 的
   比较呈现成同一序列。
5. **消融覆盖。** 核心机制没有消融行；一次变两个因素的消融；被声称必要却从未移除的
   组件。
6. **泛化范围。** 证据全部来自一个 benchmark 家族而摘要声称任务级结果；没有偏移测试集
   的分布偏移声明。
7. **可复现性表面。** 超参、数据处理、算力在某处写明（正文或点名的附录）；仓库自己的
   `training.md` 与草稿一致；venue 检查单会问而稿中缺失的任何项。
8. **声明范围越界。** 摘要或 intro 承诺超过 evaluation 展示的（"consistently"、
   "across diverse settings"）；conclusion 引入新声明；limitations 漏掉审稿人第一个
   会找到的那条。
9. **对相关工作的公道刻画。** 用最弱配置描述先前方法；漏掉显而易见的最近工作（对照
   `related_work.md` 的主题核对）；引用他人数字不给来源。
10. **图文一致。** 图与正文数字矛盾；caption 声称图并未展示的东西；坐标轴选择夸大
    效果（截断、未声明的 log）。

严重度：动摇头条声明的是 CRITICAL；任何审稿人都会列为弱点的是 MAJOR；打磨是 MINOR。
路由同对账表（缺证据 → 计划回路；措辞 → `revise`）。
